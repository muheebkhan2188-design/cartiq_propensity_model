# CartIQ: E-Commerce Purchase Propensity Machine Learning Engine

## 📌 Project Overview
CartIQ is an end-to-end predictive analytics platform that transforms raw web traffic and event logs from the Google E-Commerce dataset into behavioral intelligence. By leveraging a custom Machine Learning classification pipeline, the engine predicts the probability of a user completing a purchase during their session, enabling dynamic, real-time target marketing.

---

## 🏗️ System Architecture
The predictive pipeline operates across a modular production flow:

1. **Feature Engineering & Modeling (Python):**
   * Merges raw session traffic and user action events (`data/01_raw/`).
   * Evaluates session signals (pageviews, cart actions, duration) to train a propensity classification model.
   * Outputs a scaled probability score (`propensity_score`) for every unique user session.
2. **Database Storage Layer (PostgreSQL):**
   * Provisions optimized tables and indexing architectures via `schema.sql`.
   * Migrates scored sessions efficiently into a data warehouse database instance.
3. **Frontend Presentation (Tableau):**
   * Consumes live database tables to display predictive behavioral distributions, channel conversions, and high-intent customer segments.

---

Quick Start Guide

1. Build Virtual Environment
Install required processing and modeling libraries:
pip install -r requirements.txt

2. Initialize Target Database
Run the DDL configuration steps inside schema.sql using pgAdmin to establish the indexing arrays and production table frameworks.

3. Run the Predictive Engine
Execute the core machine learning module to ingest raw metrics, generate customer classifications, and save outputs:
python scripts/python/01_propensity_model.py

4. Open Analytics Workspace
Launch the .twbx module located within the dashboards/ directory to interface with the customer conversion matrix maps.
